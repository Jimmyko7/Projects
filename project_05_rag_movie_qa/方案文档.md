## 三、文件说明

### 3.1 离线构建阶段

**`01.电影知识库构建.py`** — 电影数据向量化脚本

```
流程：CSV 加载 → Document 构建 → 文本分割 → 向量嵌入 → Chroma 持久化

关键设计决策：
- 每部电影构建为1个 Document，包含：片名、年份、类型、导演、评分、语言、简介
- 中文分隔符优先：["\n\n", "。", "！", "？", "；", "，", "\n", " ", ""]
- chunk_size=500, chunk_overlap=50（与RAG-Agent教程一致）
- Chroma 持久化路径：./chroma_db
- collection_name="tmdb_movies"
```

**`02.电影知识库检索测试.py`** — 检索验证脚本

```
功能：
- 加载已构建的 Chroma 向量库
- 执行多种查询测试检索质量
- 显示检索结果的相关性（分数、元数据、内容片段）
- 帮助调优分块参数
```

### 3.2 在线推理阶段

**`03.RAG电影问答.py`** — 命令行 RAG 问答

```
功能：完整的命令行 RAG 问答流水线
- 检索 + 格式化上下文 + LLM 生成
- 展示完整的 LangChain LCEL 链式调用
- 支持交互式多轮问答
- 每次回答展示检索到的参考资料
```

**`04.ai_partner_rag.py`** — AI 伴侣 RAG 升级版

```
基于 07.ai_partner5.py 的增强：
✅ 保留所有原有功能（昵称、性格、会话管理、流式输出）
✅ 新增 RAG 开关（侧边栏 toggle）
✅ 启用 RAG 后自动检索电影知识库
✅ 检索结果注入系统提示词
✅ 展示检索到的电影引用
✅ 无 RAG 模式与原版行为完全一致
```

## 七、关键设计决策

### 7.1 为什么用 DashScope 嵌入而不是 DeepSeek？

DeepSeek 目前不提供独立的 Embedding API。DashScope 的 `text-embedding-v4` 在中文场景经过验证（RAG-Agent 项目已使用），且与 LangChain 有官方集成（`langchain_community.embeddings.DashScopeEmbeddings`）。

> **替代方案**：如果希望完全本地化，可以使用 Ollama 的嵌入模型（如 `nomic-embed-text`）或 `sentence-transformers`。

### 7.2 为什么混合使用 OpenAI SDK 和 LangChain？

- **LangChain** 用于 RAG 流水线（文档加载、分割、向量存储、检索），这些是 LangChain 的核心优势
- **OpenAI SDK** 用于 LLM 调用（DeepSeek），保持与现有 AI 伴侣代码一致
- 这展示了**渐进式集成**的思路——不必重写整个应用才能使用 RAG

### 7.3 为什么 AI 伴侣保留两种模式？

- **RAG 关闭**：与原版完全一致，保证向后兼容
- **RAG 开启**：自动检索电影知识库，回答更具知识性
- 用户可以在侧边栏实时切换，体验 RAG 带来的差异
